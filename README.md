# Network & Security Mini Toolkit (Python Concurrency)

โปรแกรมนี้เป็นโปรแกรมขนาดเล็กที่จำลองการทำงานของเครื่องมือทางด้าน Network และ Security เพื่อทดสอบและทำความเข้าใจการเขียนโปรแกรมแบบ Concurrency และ Parallelism ในภาษา Python

## การทำงานของโปรแกรม

โปรแกรมจะแบ่งการทำงานออกเป็น 3 ส่วน เพื่อแสดงให้เห็นถึงความเหมาะสมของแต่ละเทคโนโลยี:

### 1. Website Uptime Checker (ใช้ `Threading`)
- **การทำงาน:** ส่ง HTTP Request ไปตรวจสอบสถานะของเว็บไซต์หลายๆ เว็บพร้อมกัน
- **ทำไมถึงใช้เทคโนโลยีนี้:** การรอ Server ตอบกลับมาเป็นงานประเภท **I/O-Bound** การใช้ Threading ช่วยให้เราไม่ต้องรอเว็บแรกโหลดเสร็จ ก็สามารถส่งคำขอไปเช็คเว็บที่ 2, 3 ต่อได้เลย ทำให้ประหยัดเวลามาก

### 2. Port Scanner (ใช้ `Asyncio`)
- **การทำงาน:** สแกนพอร์ต (เช่น 80, 443, 21) บนเป้าหมายแบบดุเดือด (Concurrent)
- **ทำไมถึงใช้เทคโนโลยีนี้:** การสแกนพอร์ตต้องเปิด Socket Connection จำนวนมากๆ พร้อมกัน Asyncio ทำงานแบบ Non-blocking บน Thread เดียว ทำให้จัดการ Connection ปริมาณมหาศาล (I/O-Bound) ได้รวดเร็วและกิน RAM น้อยกว่า Threading

### 3. Hash Miner / Proof-of-Work (ใช้ `Process Pool`)
- **การทำงาน:** จำลองระบบ Proof-of-Work โดยหาค่า Nonce ที่ทำให้ได้ MD5 Hash ขึ้นต้นด้วย "00000"
- **ทำไมถึงใช้เทคโนโลยีนี้:** การเข้ารหัส (Hashing) เป็นงานที่ใช้พลังซีพียูล้วนๆ (**CPU-Bound**) การใช้ Threading หรือ Asyncio ใน Python จะติดข้อจำกัด GIL (Global Interpreter Lock) จึงต้องใช้ Process Pool เพื่อแตกโปรแกรมออกเป็นหลายๆ Process และกระจายไปทำงานบน CPU หลายๆ Cores พร้อมกัน
