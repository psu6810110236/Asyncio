# Python Concurrency & Parallelism Experiment

โปรแกรมขนาดเล็กสำหรับทดลองและทำความเข้าใจการทำงานของ **Threading**, **Asyncio**, และ **Process Pool** ในภาษา Python

## วิธีการรันโปรแกรม
1. ติดตั้ง Python (เวอร์ชัน 3.7 ขึ้นไป)
2. เปิด Terminal หรือ Command Prompt
3. รันคำสั่ง: `python main.py`

## สรุปความเข้าใจจากผลการทดลอง

ในโปรแกรมนี้ได้แบ่งการทดสอบออกเป็น 3 รูปแบบหลักๆ ดังนี้:

### 1. Threading (`threading`)
- **การทำงาน:** ใช้สร้าง Thread ใหม่ขึ้นมาทำงานพร้อมกัน เหมาะกับงานประเภท **I/O-Bound** (เช่น การดึงข้อมูลจากเว็บ, อ่านไฟล์) 
- **ผลลัพธ์:** แม้จะสั่งให้รอ (Sleep) งานละ 2 วินาที จำนวน 5 งาน แต่ใช้เวลารวมแค่ประมาณ 2 วินาทีเศษๆ เพราะทุกงานรอดำเนินการไปพร้อมๆ กัน

### 2. Asyncio (`asyncio`)
- **การทำงาน:** เป็นการทำงานแบบ Asynchronous ทำงานอยู่บน Thread เดียว แต่ใช้ระบบ Event Loop ในการสลับการทำงานเวลามีการ "รอ" (await) เหมาะกับ I/O-Bound ที่มีปริมาณเยอะมากๆ
- **ผลลัพธ์:** ใช้เวลารวมประมาณ 2 วินาทีเช่นกัน ทำงานได้ผลลัพธ์คล้าย Threading แต่ใช้ทรัพยากรระบบน้อยกว่ามาก

### 3. Process Pool (`concurrent.futures.ProcessPoolExecutor`)
- **การทำงาน:** เป็นการแตก Process ใหม่ไปยัง CPU Core อื่นๆ (Multi-core) หลุดจากข้อจำกัด GIL (Global Interpreter Lock) ของ Python เหมาะกับงาน **CPU-Bound** (เช่น การคำนวณทางคณิตศาสตร์, ประมวลผลภาพ)
- **ผลลัพธ์:** โค้ดสามารถกระจายการคำนวณ Loop หนักๆ ไปยังคอร์ต่างๆ ของ CPU ทำให้ประมวลผลเสร็จเร็วกว่าการทำแบบปกติ (Sequential)
